terraform-aws-api-gateway-v2 Architecture

HCL / Terraform AWS Infrastructure / IaC

Executive Summary

This module creates and manages AWS API Gateway V2 resources supporting both HTTP and WebSocket protocols, complete with route definitions, Lambda and HTTP integrations, JWT authorizers (Cognito/Auth0), VPC Links for private backends, custom domain names with ACM certificates, and CloudWatch access logging. It provides a fully managed, serverless API layer that scales automatically and eliminates operational overhead for building modern APIs and real-time applications.

Data Flow

  1. Client sends an HTTP request or opens a WebSocket connection to the custom domain or default endpoint.
  2. The API Gateway receives the request and matches it against the configured stage.
  3. The route key (method + path for HTTP, or $connect/$default for WebSocket) determines the target route.
  4. If the route has an authorizer, the JWT token is validated against the configured issuer and audience.
  5. The integration forwards the request to the backend: Lambda (AWS_PROXY), HTTP endpoint, or VPC Link.
  6. For VPC Link integrations, traffic traverses an ENI into the private VPC to reach internal services.
  7. The backend response flows back through the API Gateway to the client.
  8. Access logs are streamed to CloudWatch Log Groups with configurable format and 30-day retention.

Security Controls

  • JWT Authorization — Route-level JWT authorizers validate tokens from Cognito, Auth0, or any OIDC provider.
  • TLS 1.2 — Custom domains enforce TLS 1.2 security policy for all connections.
  • Disable Default Endpoint — Option to disable the default execute-api endpoint, forcing traffic through the custom domain.
  • CORS Configuration — Fine-grained CORS controls for HTTP APIs (origins, methods, headers, credentials).
  • VPC Links — Private integrations keep backend traffic within the VPC, never exposed to the internet.
  • Throttling — Stage and route-level throttling limits protect against abuse and runaway costs.
  • CloudWatch Logging — Detailed access logs for audit, debugging, and compliance.

Production Readiness Checklist

  • Configure JWT authorizers on all sensitive routes
  • Set up a custom domain with ACM certificate
  • Disable the default execute-api endpoint (disable_execute_api_endpoint = true)
  • Enable access logging with CloudWatch destination ARN
  • Configure throttling limits on stages and routes
  • Use VPC Links for internal/private backend services
  • Set restrictive CORS policies (avoid wildcard origins)
  • Enable auto-deploy for continuous deployment workflows
  • Configure Route 53 CNAME/alias for the custom domain target
  • Monitor API metrics via CloudWatch detailed metrics

Deployment

Quick Start

module "api_gateway" {
  source = "github.com/kogunlowo123/terraform-aws-api-gateway-v2"

  name          = "my-http-api"
  description   = "My HTTP API"
  protocol_type = "HTTP"

  routes = {
    "get-hello" = {
      method          = "GET"
      path            = "/hello"
      integration_uri = aws_lambda_function.hello.invoke_arn
    }
  }

  stages = {
    "$default" = {
      auto_deploy = true
    }
  }
}
